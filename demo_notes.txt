Demo Notes — TakeMeter (AI201 Project 3)
Target length: 2–3 minutes

WHAT TO HAVE ON SCREEN (tab order)
1. README.md or planning.md (intro + labels)
2. data/r_nba_labeled.csv (data section — scroll only)
3. baseline_evaluation.json (baseline section)
4. evaluation_results.json (fine-tuning + closing)
5. confusion_matrix.png (matrix + example error)
6. Optional: planning.md Section 3 (edge cases) when discussing errors

────────────────────────────────────────
1. INTRO (~20 sec)
────────────────────────────────────────

Say:
"Hi, I'm Quonlee Howery. This is TakeMeter, my AI201 Project 3 for CodePath.

I built a text classifier for r/nba — Reddit's main NBA community. The goal is to tell apart four types of posts: analysis, hot takes, news updates, and meta or community content.

The idea is that during playoffs or trade deadline, a feed could boost real analysis and deprioritize low-effort hot takes."

Show: README title or planning.md Sections 1–2.

────────────────────────────────────────
2. LABELS (~30 sec)
────────────────────────────────────────

Say:
"I defined four labels based on what the community already cares about:

- analysis — posts backed by stats, film, or structured reasoning
- hot_take — bold opinions without real evidence
- news_updates — injury reports, highlights, trade news
- meta_community — fandom, media, or subreddit stuff like SERIOUS threads

The tricky boundary is hot take versus analysis. One stat used to support an extreme ranking is still a hot take — I documented that primary-move rule in planning.md Section 3."

Show: planning.md label table (Section 2) or README label list.

────────────────────────────────────────
3. DATA (~20 sec)
────────────────────────────────────────

Say:
"I collected 300 public r/nba posts using the PullPush API, labeled them manually with help from a rule-based script I reviewed, and split 70-15-15 for train, validation, and test. The test set is 45 posts — 12 analysis, 9 hot takes, 14 news, 10 meta — that neither model saw during training."

Show: data/r_nba_labeled.csv (scroll briefly — don't read rows aloud).

────────────────────────────────────────
4. BASELINE (~25 sec)
────────────────────────────────────────

Say:
"For Milestone 4 I ran a zero-shot Groq baseline using llama-3.3-70b-versatile with a prompt built from my label definitions.

It got 73.3% accuracy on the test set — 45 out of 45 responses parseable. It was good at news and meta, but hot_take recall was only 22% — it missed most opinion posts and often lumped them into analysis or community."

Show: baseline_evaluation.json — point to baseline_accuracy and hot_take recall in classification_report.

────────────────────────────────────────
5. FINE-TUNING (~35 sec)
────────────────────────────────────────

Say:
"For Milestone 5 I fine-tuned DistilBERT on 210 training examples.

My first run actually did worse than the baseline — 71.1% — because the model never predicted hot_take at all. It collapsed to mostly news.

I fixed that with class-weighted loss and five epochs instead of three. Test accuracy came up to 82.2% — about 9 points over baseline, beating my plus-8 target. Hot_take recall jumped to 89%, though analysis recall is still only 50% — that's the tradeoff."

Show: evaluation_results.json — finetuned_accuracy, improvement, and hot_take vs analysis recall.

────────────────────────────────────────
6. CONFUSION MATRIX (~25 sec)
────────────────────────────────────────

Say:
"Here's the confusion matrix — rows are true labels, columns are predictions.

News and meta are nearly perfect. The main error block is analysis misclassified as hot_take — four posts in that cell. That's the off-diagonal I planned for: stats present, but the post's main move is still a sweeping comparison, not step-by-step reasoning."

Show: confusion_matrix.png
Point at: row analysis → column hot_take (the cell with 4).
Optional: also note analysis recall 6/12 in evaluation_results.json.

────────────────────────────────────────
7. EXAMPLE ERROR (~25 sec)
────────────────────────────────────────

Say:
"One test-set post in that bucket was labeled analysis but the model called hot_take:

'SGA and Jdub finish the series with more assists individually and less turnovers combined than Jokic — SGA 46 assists and 14 turnovers, Jdub 42 and 10, Jokic 41 and 30.'

It's a stat table supporting an implicit ranking — SGA was better than Jokic this series. I labeled it analysis because the numbers are structured, but the model read it as opinion-with-stats — exactly the boundary in planning.md."

Show: evaluation_results.json or confusion_matrix.png while paraphrasing; optional scroll to that row in r_nba_labeled.csv if you want the full text on screen.

────────────────────────────────────────
8. CLOSING (~15 sec)
────────────────────────────────────────

Say:
"TakeMeter met my success criteria: beat the baseline by 8.9 points, every class hit F1 above 0.6 — analysis was the tightest at 0.63 — and no single label dominated predictions.

Thanks for watching."

Show: README results table or submission checklist (demo link already filled in).
